# 本工程的 Claude 协作规则

> 给 Claude / 其它 agent 接管本工程时遵循的工作纪律。

## 规则

1. **每次任务完成要用精简的语言记录日志**
   - 落在 `LOG.md`，每条 1–3 行：日期 + 做了什么 + 当前状态。
   - 不写情绪化语言、不抄代码、不复述文档已有内容。

2. **遇到的错误、bug 等的修复经验都要详细记录**
   - 落在 `BUGFIX.md`，按"现象 → 根因 → 修复 → 复发预防"四段写。
   - 越是踩过的坑越要写清楚，避免下一次重蹈。

3. **每完成一个任务都要更新 `TODO.md`**
   - 完成项立即从 TODO 中删除或合并到一个简短的"近期完成"小节；
   - 该文件常年不能超过两屏：定期清理"近期完成"中陈旧的条目。

4. **更新工程后也要立即更新 PROJECT_GUIDE、IMPLEMENTATION_NOTES 等所有相关文档**

5. **不论任务成功或失败，都要按上述要求更新文档**

6. **参数化优先**
   - 任何来自论文的尺寸/位宽/层数/通道数都写成 `parameter` / `localparam`，
     默认值取自 `2025_TCASI_VLSI_Compact_Shortcut_DAE` 论文。
   - 工程中尽量不要硬编码数字。

7. **IP 核处理规则**（严格执行）
   - 论文里凡是依赖 Xilinx / 厂商 IP 核的模块（DSP48E1、Block Memory Generator、AXI Lite Slave、
     AXI DMA、PCIe、FFT 等），在**对应 RTL 文件头部**用注释写明：
     ```
     // [IP] <IP 名称> — 出现在论文 <Fig./Sec.> 处。
     //   行为级实现：YES / NO。
     //   YES → 工程内已自实现；综合时可由 Vivado 推断或显式替换为 IP。
     //   NO  → 此文件仅为接口 stub；下方 ports 定义即推测的对外接口。
     ```
   - 如果 IP 的功能用 Verilog 行为级**可以**自己实现（比如 BRAM、DSP MAC），就**自行实现**，
     并在文件头标注"综合时建议替换为 IP <名称>，详见 IMPLEMENTATION_NOTES Step 6"。
   - 如果**无法**自己实现（比如 AXI 总线协议、PCIe IP、PLL/MMCM、ADC IP），就：
     1. 推测它的接口 (port list) 写成 `*_stub.v`；
     2. 文件**头部**注释写明：① 该 IP 在论文哪个 Fig./Sec.；② 端口含义；③ 握手时序；
        ④ 典型使用顺序（"先拉这个再读那个"那种 step list）；
     3. 在 `IMPLEMENTATION_NOTES.md` 对应 Step 里加一条"上板时用 Xilinx IP 替换"。
   - 本工程目前涉及的 IP 列表见 `IMPLEMENTATION_NOTES.md` "IP 核清单"小节，要保持同步更新。

8. **所有文档必须"简单、详细、易懂"，面向 Verilog 初学者**（严格执行）
   - 假定读者：只用 Verilog 写过 SPI / UART / 简单状态机的初级使用者，**没有**写过任何
     深度学习加速器、CNN、量化、Pixel Shuffle、AXI、DMA 的经验。
   - 所有 markdown 文档（PROJECT_GUIDE / IMPLEMENTATION_NOTES / TODO / LOG / BUGFIX 等）
     必须满足：
     1. **先打比方再讲术语**：每引入一个新概念（如"卷积"、"shortcut"、"量化"、
        "Pixel-Unshuffle"）先用一两句生活化比喻解释，再给学术定义。
     2. **每个模块要回答四件事**：① 是什么 ② 解决什么问题 ③ 接口/端口含义
        ③ 跟同类模块/上下游的连接关系。
     3. **每条数据通路要"跟着一个像素/一拍走一遍"**：用具体数值示例代替抽象描述。
     4. **不堆术语缩写**：第一次出现的英文缩写一定先写全称，再加一句解释。
     5. **图优于文字、表优于段落**：能用 ASCII 图、表格表达的就不要用大段文字。
     6. **不要省略"为什么这么做"**：尤其论文式 (5)–(14) 这类数学，写代码注释里要给
        一句"翻译成人话"。
   - 编辑/新增文档前先想：**"我是只写过 SPI 的人，看到这一段我能不能立刻明白？"** 不
     能 → 重写。
